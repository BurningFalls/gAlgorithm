# Info
[비부분수열]](https://boj.kr/1885)

## 💡 풀이 방법 요약

최대 100,000 개가 들어오고, 한 줄마다 입력이 들어오면 이런 형태는 보통 들어오는 순서대로 확인해도 문제를 해결할 수 있다는 뜻입니다.

그렇지 않다면 입력 오버헤드가 크거든요. 물론 아닐수도 있으니 신봉하진 맙시다.

결국 해결한 풀이는, 앞에서부터 1 부터 k가 모두 들어있는 부분수열이 몇 개가 있느냐 입니다.

1 부터 k까지 모든 수가 있는 부분배열만이, 1 부터 k 를 조합하여 만들 수 있는 부분수열을 만드는 조건이기 떄문입니다.

따라서 앞에서부터 해당 부분수열이 몇 개 존재하는 지 세고, 그 센 갯수보다 하나 더 긴 부분수열은 필시 부분수열로 존재하지 않게 됩니다.

## 👀 실패 이유

첫 시도는 도저히 방법이 생각이 나질 않아 진짜 만들 수 있는 부분수열을 다 만들어 줬습니다.

당연히 메모리 초과로 터졌고, 두 번째는 1 부터 k 까지의 입력중 각 숫자가 몇 개 나왔는지 확인해서 (가장 적은 숫자가 나온 개수 + 1) 로 생각했는데, 역시나 11퍼였나 에서 틀렸습니다.
